## ✨ Возможности

### 🔬 Анализ растений
| Функция | Описание |
|---------|----------|
| 🎯 **YOLO-сегментация** | Разделение на корни / стебель / листья с настраиваемыми порогами уверенности |
| 🕸️ **Скелетизация** | Преобразование масок в одно-пиксельные скелеты (Zhang-Suen / Lee) |

## 🛠 Установка

### Требования
- Python **3.9–3.11**
- ОЗУ: **8+ ГБ** (16+ ГБ рекомендуется для больших изображений)
- GPU: **опционально** (CUDA 11.8+ для ускорения YOLO)

Скачать веса best_h.pt — после скачивания положи файл в папку ниже.
https://disk.yandex.ru/d/rBLM4rVSazp5Kg


### 1. Клонирование репозитория
```bash
git clone https://github.com/yourusername/plant-graph-analyzer.git
cd plant-graph-analyzer
```

### 2. Создание виртуального окружения
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

Или вручную:
```bash
pip install fastapi uvicorn[standard] opencv-python-headless numpy \
            networkx scikit-image scipy matplotlib ultralytics \
            python-multipart pillow
```

### 4. Подготовка модели YOLO
```
plant-graph-analyzer/
└── yolo_weights/
    └── plants_optimized_seg/
        └── weights/
            └── best.pt  ← ваша обученная модель
```

> 💡 **Нет модели?** Анализ будет работать в режиме "только морфология" (без YOLO).

## ⚙️ Конфигурация

### Файл `server.py` — основные настройки

```python
DEFAULT_CONFIG = {
    # Пути
    "MODEL_PATH": "./yolo_weights/plants_optimized_seg/weights/best.pt",
    
    # YOLO параметры
    "YOLO_CONF_ROOT": 0.001,    # Порог уверенности для корней
    "YOLO_CONF_STEM": 0.10,     # Для стебля
    "YOLO_CONF_LEAF": 0.30,     # Для листьев
    "YOLO_IMG_SIZE": 1280,      # Размер входа модели (640/1280/1920)
    
    # Граф
    "GRAPH_CONNECT_RADIUS": 1.7,  # Радиус соединения узлов скелета
    "REMOVE_SMALL_COMP_SIZE": 30, # Удалять компоненты < N узлов
    
    # Соединение корней
    "ROOT_CONNECT_MAX_DIST": 50,  # Макс. дистанция соединения (px)
    "ROOT_ANCHOR_MAX_DIST": 50,   # Дистанция до якоря для очистки
    
    # Морфология
    "MORPH_KERNEL_SIZE": 3,       # Размер ядра морф. операций
    "COMBINED_MIN_SIZE": 60,      # Мин. площадь объекта (px²)
    
    # Калибровка
    "SQUARE_SIZE_MM": 25.0,       # Размер квадрата шахматки (мм)
    "CHESSBOARD_SIZE": (7, 4),    # Внутренние углы (не квадраты!)
    
    # Отрисовка
    "EDGE_THICKNESS": 2,          # Толщина линий графа
    "NODE_END_RADIUS": 5,         # Радиус конечных узлов
    "NODE_MID_RADIUS": 2,         # Радиус промежуточных узлов
    
    # Цвета (BGR)
    "COLORS": {
        'root': {'edge': (255, 0, 180), 'node_end': (0, 0, 255)},
        'stem': {'edge': (0, 220, 0),   'node_end': (0, 100, 0)},
        'leaf': {'edge': (0, 180, 255), 'node_end': (0, 80, 180)},
    },
}
```

### Переопределение параметров через фронтенд
Все параметры из `DEFAULT_CONFIG` можно изменить в интерфейсе — они автоматически передаются на сервер.

## 🌐 Использование

### 🔹 Шаг 1: Калибровка (рекомендуется)
1. Сфотографируйте шахматную доску **перпендикулярно** камере
2. Перетащите фото в блок **"📐 Калибровка"**
3. Укажите:
   - Количество квадратов по горизонтали/вертикали
   - Размер одного квадрата в мм
4. Нажмите **"🚀 Запустить калибровку"**
5. Дождитесь визуализации и запомните масштаб: `📏 4.23 px/mm`

> 💡 **Важно:** Указывайте количество **квадратов**, а не углов!  
> Для доски 8×5 квадратов → внутренние углы = **7×4** (автоматически вычисляется).

### 🔹 Шаг 2: Анализ растения
1. Перетащите фото растения в блок **"🔬 Анализ"**
2. При необходимости настройте параметры:
   - `YOLO_CONF_*` — если детектирует слишком много/мало
   - `GRAPH_CONNECT_RADIUS` — если граф фрагментирован
   - `ROOT_ANCHOR_MAX_DIST` — если удаляются нужные корни
3. Нажмите **"🔍 Запустить анализ"**
4. Изучите результат:
   - 🖼️ Визуализация с наложенным графом
   - 📊 Таблица с площадью и длиной для каждой части
   - 📏 Длина в см (если проведена калибровка)

### 🔹 Шаг 3: Экспорт результатов
1. Нажмите **"📥 Экспорт результатов"**
2. Выберите формат: JSON / CSV / GraphML
3. Файл скачается автоматически

| `YOLO_CONF_LEAF` | 0.01–1.0 | 0.30 | Порог для листьев |

| Параметр | Диапазон | По умолчанию | Описание |
|----------|----------|--------------|----------|

| `REMOVE_SMALL_COMP_SIZE` | 1–500 | 30 | Удалять компоненты с < N узлами |
| `GRAPH_PRUNE_BRANCHES` | 0–50 | 5 | Обрезать ветви короче N узлов |

## 📐 Калибровка масштаба

### Как подготовить шахматную доску
1. Распечатайте шахматку с **известным размером квадрата** (например, 25×25 мм)
2. Наклейте на жёсткую основу (картон, пластик)
3. Сфотографируйте:
   - Камера **перпендикулярно** доске
   - Хорошее **равномерное освещение**
   - Доска занимает **~50–80%** кадра
   - Без сильных бликов и размытия

### Расчёт внутренних углов
```
Количество квадратов:  8 × 5
Количество углов:     (8-1) × (5-1) = 7 × 4  ← указывать в настройках!
```

### Проверка качества калибровки
| Метрика | Хорошо | Тревога |
|---------|--------|---------|
| **RMS ошибка** | < 0.3 | > 1.0 |
| **pixels_per_mm** | 3–10 px/mm | < 1 или > 20 |
| **Визуализация** | Углы точно на пересечениях | Углы смещены, дисторсия не устранена |

### Повторная калибровка
Калибровочные данные сохраняются в:
```
calibration_data_single.npz
```

Чтобы перекалибровать:
1. Удалите файл `calibration_data_single.npz`
2. Загрузите новое фото шахматки
3. Запустите калибровку заново

## 🐛 Устранение проблем

### ❌ Ошибка: `ValueError: [TypeError("'numpy.float32' object is not iterable")]`
**Причина:** FastAPI не может сериализовать NumPy-типы в JSON.  
**Решение:** Убедитесь, что все `return` обернуты в `to_json_serializable()`.

### ❌ Ошибка: `module 'cv2' has no attribute 'TERM_CRITERIA_MAX_ITER'`
**Причина:** Кириллическая буква в названии константы (частая ошибка при копировании).  
**Решение:** Перепишите строку вручную латинскими буквами:
```python
# ✅ Правильно:
criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
```

### ❌ Шахматка не находится
**Возможные причины:**
- Неправильное количество углов (указали квадраты вместо углов)
- Плохое освещение / блики / размытие
- Доска под углом > 30°
- Неправильный порядок (колонки, строки)

**Решение:**
```python
# Добавьте отладку в calibrate_single_image():
print(f"🔍 Ищем {CB[0]}×{CB[1]} углов")
print(f"📐 Изображение: {gray.shape[::-1]}")
```

### ❌ Корни удаляются слишком агрессивно
**Причина:** Слишком маленький `ROOT_ANCHOR_MAX_DIST` или нет якорей (стебель/листья не детектированы).  
**Решение:**
1. Увеличьте `ROOT_ANCHOR_MAX_DIST` до 80–100
2. Проверьте детекцию стебля/листьев (снизьте `YOLO_CONF_STEM/LEAF`)
3. Включите `SAVE_INTERMEDIATE: "all"` для отладки масок

### ❌ Медленная обработка
**Оптимизация:**
```python
# В DEFAULT_CONFIG:
"YOLO_IMG_SIZE": 640,           # Уменьшить вход модели
"GRAPH_MAX_NODES": 20000,       # Ограничить узлы графа
"USE_GPU": "true",              # Включить GPU если есть
"MAX_WORKERS": 2,               # Ограничить потоки на слабых CPU
```

### ❌ Изображение результата не грузится в браузере
**Причина:** FastAPI не раздаёт статические файлы.  
**Решение:** Убедитесь, что в `server.py` есть:
```python
app.mount("/results", StaticFiles(directory=os.path.join(BASE_DIR, "results")), name="results")
```
