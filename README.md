# Сегментация растений — веб, API и Telegram-бот

Веб-интерфейс и бот для анализа изображений растений: сегментация (корень, стебель, лист), площади и длины в мм/см. Используется стандартная калибровка (фиксированный масштаб).

---

## Что нужно для запуска

- **Python 3.10+**
- **PyTorch** (желательно с CUDA для скорости)
- **Веса модели YOLO** — файл `best.pt` (см. раздел ниже)

---

## Веса модели YOLO (обязательно)

В репозитории **нет файла весов** — он слишком большой для GitHub (>100 MB). Без весов бекенд не запустится.

### Скачать веса (для тех, кто клонировал проект)

**Ссылка на веса:** вставь сюда ссылку на скачивание (Google Drive, Яндекс.Диск и т.п.) и сохрани README.

- **[Скачать веса best_h.pt](ССЫЛКА_СЮДА)** — после скачивания положи файл в папку ниже.

**Куда положить файл после скачивания:**

1. В корне проекта создай папки (если их нет):  
   `all_data/yolo_weights/plants_optimized_seg/weights/`
2. Положи скачанный файл в эту папку с именем **`best.pt`** или **`best_h.pt`** (оба варианта поддерживаются).
3. Запускай бекенд — веса подхватятся автоматически.

Пример полного пути (Windows):  
`proj\all_data\yolo_weights\plants_optimized_seg\weights\best_h.pt`

**Если ты автор репозитория:** загрузи свой файл весов (например, `best_h.pt`) на Google Drive / Яндекс.Диск, получи ссылку на скачивание и замени в этом разделе текст `ССЫЛКА_СЮДА` на реальную ссылку, чтобы другие могли скачать веса одной кнопкой.

**Другие варианты:** обучить свою модель по разметке в `all_data/` или положить свои веса YOLO-seg (корень/стебель/лист) в ту же папку как `best.pt` или `best_h.pt`.

## Установка (один раз)

### 1. Перейди в папку проекта

```bash
cd путь/к/проекту/proj
```

Например в PowerShell (Windows):

```powershell
cd "c:\Users\leonid\Downloads\проект\proj"
```

### 2. Создай виртуальное окружение (рекомендуется)

**Windows (PowerShell):**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**Linux / macOS:**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установи зависимости

```bash
pip install -r requirements.txt
```

Если PyTorch ещё не установлен:

```bash
pip install torch torchvision
```

(На Windows с NVIDIA GPU можно взять вариант с CUDA с сайта PyTorch.)

## Запуск

### Шаг 1: Запуск бекенда (обязательно первым)

Из корня проекта (`proj`):

**Windows (PowerShell):**

```powershell
uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

**Linux / macOS:**

```bash
uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

Проверка: открой в браузере [http://127.0.0.1:8000/health](http://127.0.0.1:8000/health) — должен быть ответ `{"status":"ok"}`.

### Шаг 2: Веб-интерфейс

Открой в браузере:

**http://127.0.0.1:8000**

Либо открой файл `frontend/index.html` (двойной клик) — страница обращается к бекенду по адресу выше.

Дальше: загрузка фото растения → при необходимости настройки предобработки → «Запустить анализ» → просмотр результата и таблицы.

### Шаг 3: Telegram-бот (по желанию)

1. Создай бота в Telegram через [@BotFather](https://t.me/BotFather) и скопируй токен (или используй свой, если уже создан).
2. Запусти бота (бекенд должен быть уже запущен). Токен задаётся переменной окружения.

**Windows (PowerShell):**

```powershell
$env:TELEGRAM_BOT_TOKEN = "8668919101:AAGsGeIxvea4W17FuIUZp15wy5KACy9Q0E4"
python -m telegram_bot.bot
```

**Linux / macOS:**

```bash
export TELEGRAM_BOT_TOKEN="8668919101:AAGsGeIxvea4W17FuIUZp15wy5KACy9Q0E4"
python -m telegram_bot.bot
```

**Важно:** если репозиторий публичный, не храни токен в README — отзови этот токен в @BotFather, создай нового бота и используй только переменную окружения (не коммить токен в репозиторий).

В боте: отправь фото растения для анализа; команда `/settings` — настройки предобработки (ч/б, контраст, яркость).

## Калибровка (масштаб в мм/см)

Используется **только стандартная калибровка** — фиксированный масштаб. Значение задаётся переменной окружения `MM_PER_PX` (по умолчанию подобрано под типовую съёмку). Результаты выдаются в мм²/см² и мм/см.

```

│   └── main.py          # FastAPI: анализ, калибровка, раздачи файлов

├── requirements.txt

```

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather (обязателен для бота). Пример в разделе «Запуск» выше. |
| `PLANT_API_BASE` | Адрес бекенда (по умолчанию `http://127.0.0.1:8000`). |
| `MM_PER_PX` | Масштаб «стандартной» калибровки (мм на пиксель). |
